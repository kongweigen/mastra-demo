# 场景去重策略

## 合并条件

- **相同地点不同描述** → 合并为最完整描述（如两集都有"咖啡厅"但描述不同）
- **同义名称** → 合并为更常用名称（"客厅"="起居室"）
- **包含关系** → 视上下文决定（"咖啡厅"和"星巴克咖啡厅"可能是同一个）
- **名称+括号注释** → 合并，去掉括号（"极寒雪地（梦境）"→"极寒雪地"）
- **距离相近但有戏的空间** → 保持独立（"门口"和"门廊"如果都有动作/对话，不合并）
- **完全同义** → "门口"和"门廊"、"楼道"和"楼梯间"在同一栋建筑时视为同一场景

## 不合并

- 明确不同地点（"林家客厅" vs "张家客厅"）
- 相同类型不同实例（"医院急诊室" vs "医院病房"）
- 空间层级不同（"医院大厅" vs "医院病房" 是两个场景）
- 不同 scene_type（"车内" vs "路边" 不合并）

## 字段合并规则

| 字段 | 合并策略 |
|------|---------|
| name | 取更简洁、更常用的名称 |
| scene_type | 取一致值；冲突时取出现次数多的 |
| mood | 合并所有出现的情绪，用顿号分隔（如"温馨、紧张"）|
| environment_desc | 合并为最完整的描述，去重复内容 |
| time_desc | 取最典型/最常出现的时间 |
| location_desc | 取最详细的描述 |

## 合并示例

```
第1集: {"name": "极寒雪地（前世记忆）", "scene_type": "OUTDOOR", "mood": "绝望", "environment_desc": "零下八十度"}
第2集: {"name": "极寒雪地", "scene_type": "OUTDOOR", "mood": "孤寂", "environment_desc": "漫天大雪，冰冷死寂"}

合并后: {"name": "极寒雪地", "scene_type": "OUTDOOR", "mood": "绝望、孤寂", "environment_desc": "零下八十度，漫天大雪，冰冷死寂"}
nameMapping: {"极寒雪地": "极寒雪地", "极寒雪地（前世记忆）": "极寒雪地"}
```

## 场景名规范化

合并时统一场景名格式：
- 去掉括号注释："林家卧室（温馨）"→"林家卧室"
- 去掉集数信息："第3集的咖啡厅"→"咖啡厅"
- 保持简洁：后续分镜拆解需要精确匹配场景名

## 输出格式

```json
{
  "scenes": [
    {"name": "极寒雪地", "scene_type": "OUTDOOR", "mood": "绝望、孤寂", "environment_desc": "零下八十度，漫天大雪...", "time_desc": "夜晚", "location_desc": "大雪封山的山林"}
  ],
  "scene_name_mapping": {
    "极寒雪地": "极寒雪地",
    "极寒雪地（前世记忆）": "极寒雪地",
    "末日冰原": "极寒雪地"
  }
}
```
