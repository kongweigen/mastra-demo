# 原创剧本创作

## 适用场景

当用户要求"帮我写一个剧本"、"写一个500字的都市短剧"、"创作一个悬疑故事"等从零创作内容的需求时，加载此技能。

## 核心原则

你本身就是一个 AI 语言模型，拥有创作能力。当用户要求原创内容时，你应该**直接创作**，而不是说"没有对应工具"。

## 创作流程

### Step 1: 理解需求

从用户消息中提取：
- **题材/类型**: 都市、古装、悬疑、甜宠、逆袭等
- **字数要求**: 默认 500-800 字（短剧适合简短紧凑）
- **风格偏好**: 第一人称/第三人称、节奏快慢、情感基调
- **特殊要求**: 角色数量、场景限制、特定情节

如果用户需求太模糊（如只说"写个剧本"），先追问类型和基本设定。

### Step 2: 创作剧本

按短剧格式创作，包含：
- **标题**: 吸引眼球的短剧标题
- **角色设定**: 2-5 个角色，标注性别、年龄、性格
- **场景**: 2-4 个主要场景
- **剧情**: 分场景的对话和动作描写

短剧剧本格式规范：
```
【场景一：地点/时间】

角色A（动作/表情）
"台词内容"

旁白：叙述内容

角色B（动作/表情）
"台词内容"
```

### Step 3: 自动创建项目

创作完成后，调用 `createProject` 工具将剧本内容存入项目：

```
createProject(
  userIdStr = 从上下文 [userId=xxx] 获取,
  name = 剧本标题,
  novelIdStr = "",          // 原创不关联小说
  textContent = 完整剧本内容,  // ★ 关键：把创作的剧本作为 textContent 传入
  aspectRatio = "9:16",     // 短视频默认竖屏
  stylePresetIdStr = ""
)
```

### Step 4: 引导下一步

项目创建成功后，引导用户进入生产流程：
- 提取角色和场景
- 进行剧本二创改写（润色优化）
- 拆解分镜

## 字数指导

| 用户要求 | 实际创作 | 说明 |
|---------|---------|------|
| 未指定 | 500-800字 | 短剧默认长度，紧凑有力 |
| 500字 | 400-600字 | 允许上下浮动 |
| 1000字+ | 800-1200字 | 短剧上限，超过建议分集 |
| 多集/长篇 | 每集500-800字 | 用换行或分隔符标记分集点 |

## 创作质量要求

1. **冲突前置**: 开篇30秒内建立核心冲突
2. **对话为主**: 短剧以对话推进，减少大段旁白
3. **节奏紧凑**: 每个场景都有信息推进，不要注水
4. **角色鲜明**: 用台词和行动塑造角色，避免标签化描述
5. **结尾有力**: 反转、悬念、情感高潮，让观众想看下一集

## 关键规则

1. **能创作就创作**: 不要推脱说"我只能调度"，你有创作能力
2. **创作后必建项目**: 写完剧本后必须调用 createProject 存入系统
3. **textContent 传完整内容**: 不要截断或摘要，把完整剧本作为 textContent
4. **引导后续流程**: 创建项目后立即给出下一步操作按钮
