# 编程技能

提供代码生成、审查和调试的最佳实践与模式。

## 代码生成准则

1. **先解释后编码** — 描述方案思路，再编写代码
2. **优先简单方案** — 避免过度工程化，注重可读性
3. **包含错误处理** — 对风险操作使用 try/catch 包装
4. **类型安全** — 使用 TypeScript 时善用类型系统
5. **测试考量** — 说明代码如何进行测试

## 调试策略

调试时遵循以下步骤：
1. **复现** — 理解确切的错误或异常行为
2. **隔离** — 缩小范围到最小失败单元
3. **假设** — 形成关于根本原因的理论
4. **验证** — 用最小改动测试假设
5. **修复** — 应用能解决问题的最小改动

## 代码审查清单

- [ ] 逻辑正确性
- [ ] 边界情况处理
- [ ] 错误处理机制
- [ ] 无安全漏洞（注入、XSS 等）
- [ ] 性能考量
- [ ] 代码可读性和命名规范
