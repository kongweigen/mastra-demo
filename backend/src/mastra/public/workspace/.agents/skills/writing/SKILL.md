# 写作技能

提供不同格式下有效书面沟通的指导方针。

## 核心原则

1. **清晰优于花哨** — 写作是为了让人理解，而非炫技
2. **主动语态** — "函数返回一个值"，而非"一个值被返回"
3. **具体明确** — "延迟降低了 40ms"，而非"性能有所提升"
4. **一段一主题** — 保持段落聚焦
5. **重要信息前置** — 将最关键的观点放在最前面

## 分格式指南

### 技术文档
- 以一句话总结该事物的功能开头
- 在前 3 段内包含一个快速上手示例
- 使用带语言标注的代码块
- 链接到相关概念，而非重复解释
- 包含"常见陷阱"章节

### 博客文章
- 前 2 句话要抓住读者注意力
- 每 3-4 段使用小标题
- 至少包含一个具体案例或实际示例
- 以明确的行动号召或要点总结结尾

### 邮件沟通
- 主题行 = 所需操作 + 主题
- 第一行 = 核心信息或请求
- 多个事项使用要点列表
- 明确的下一步行动及截止时间
- 控制在 5 段以内

## 编辑清单
- [ ] 删除填充词（非常、真的、其实、基本上）
- [ ] 缩短超过 50 字的长句
- [ ] 面向混合受众时用通俗语言替代术语
- [ ] 检查时态和语态的一致性
- [ ] 验证所有论断有依据支持
