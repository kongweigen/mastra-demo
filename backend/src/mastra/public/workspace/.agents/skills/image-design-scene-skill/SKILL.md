# 场景多角度参考网格图 Prompt 工程规范

为单个场景生成一张 1x3 横排网格图，三格分别展示全景远景、中景细节、特写局部，生成后由后端裁剪为三张独立参考图。

用途：建立项目场景视觉基调，后续分镜画面风格将参考这些场景图。

## 布局规则（固定 1x3）

固定 1 行 3 列，无需根据数量动态调整：

| 格位 | 位置 | 视角 | 说明 |
|------|------|------|------|
| 第1格 | 左 | 全景远景 | 展示场景整体空间结构、纵深层次 |
| 第2格 | 中 | 中景 | 从另一个角度展示主要区域的细节和陈设 |
| 第3格 | 右 | 特写局部 | 场景标志性元素或氛围细节的近距离特写 |

禁止其他布局（如 3x1 竖排、2x2 等），裁剪程序按 1行3列 等宽切分。

## Prompt 结构

```
[布局指令] + [第1格：全景描述] + [第2格：中景描述] + [第3格：特写描述] + [一致性] + [风格]
```

### 布局指令（固定前缀）

每条提示词必须以此开头：
```
一张1行3列的场景多角度参考网格图，共3个画面格，用粗黑分隔线均匀分隔。
```

### 逐格内容

三格共用同一场景的环境设定（色调、光线、天气、氛围），只在构图和重点上有区别。

**第1格（左）— 全景远景**
```
第1格（左）：宽幅全景远景镜头，[场景类型和位置]。[前景细节]，[中景主体]，[背景/远景]，[光线]，[氛围]。
```

**第2格（中）— 中景/侧面角度**
```
第2格（中）：中景镜头，从另一角度展示同一场景。[该角度可见的主要区域]，[家具/陈设/植被等细节]，[光线在此角度的表现]。
```

**第3格（右）— 特写局部**
```
第3格（右）：特写镜头，聚焦场景中的标志性元素。[具体物品/装饰/材质的近距离描写]，[光线在物体上的质感表现]。
```

### 描述层级

每格的场景描述按以下层级组织：

1. **空间定位**: 场景类型 + 具体位置（如 `现代高档公寓客厅内部`）
2. **环境分层**: 前景 → 中景 → 背景（全景格），主要区域细节（中景格），标志物特写（特写格）
3. **时间天气**: 黄金时段 / 阴天午后 / 雨夜 / 深夜 等
4. **光线**: 最影响氛围的要素（自然光/灯光/混合光的方向和色温）
5. **氛围**: 温馨 / 压抑 / 繁忙 / 宁静 / 忧郁 等

注意：第2格中景和第3格特写需要补充该视角特有的细节，而非简单重复全景描述。

### 一致性约束

```
三格中同一场景的色调、光线、天气、陈设完全一致，仅拍摄角度和景别不同，画面中无人物。
```

### 风格后缀

使用请求上下文中的「项目风格」字段作为风格描述。如果项目风格为"未设置"，默认使用"电影写实风格"。

```
{项目风格}，建筑摄影，画面中无人物，整洁网格布局，粗黑分隔线清晰，超高细节，8K画质。
```

## 输出格式（严格 JSON，不加 Markdown 代码块）

```json
{
  "scenes": [
    {
      "id": 场景ID,
      "prompt": "一张1行3列的场景多角度参考网格图...",
      "negativePrompt": "模糊, 低质量, 画面中有人物, 水印, 格子合并, 分隔线不清晰, 三格场景不一致",
      "promptParams": {"grid_rows": 1, "grid_cols": 3, "ratio": "16:9"}
    }
  ]
}
```

- 每个场景独立一条记录
- `grid_rows` 固定为 1，`grid_cols` 固定为 3
- `ratio` 固定为 `"16:9"`（横向宽图）
- `promptParams` 是裁剪程序的契约，必须准确
- 只输出 JSON，不要添加任何解释文字
- 所有提示词使用中文

## 检查清单

- 布局必须为 1行3列（grid_rows=1, grid_cols=3）
- ratio 为 "16:9"
- 第1格全景远景、第2格中景侧面、第3格特写局部，顺序不可变
- 三格场景色调/光线/天气/陈设完全一致
- 每格都有该景别的专属细节（不是简单重复）
- 全景格：前景/中景/背景分层
- 中景格：另一视角的空间细节
- 特写格：标志性物品/材质的近距离描写
- 必须包含"画面中无人物"
- 中文提示词
- ID 与输入完全对应
