# 资产质量评审标准

本技能定义各维度的评分标准和关卡决策阈值，帮助 QualityGateCoordinator 做出一致的质量判断。

## 评审维度 (0.0 - 1.0)

### 1. 完整性 (completeness)

资产是否覆盖所有分镜。

| 分数 | 条件 |
|------|------|
| 1.0 | 所有分镜都有对应资产 |
| 0.8 | 缺失 ≤ 10% |
| 0.5 | 缺失 10% - 30% |
| 0.0 | 缺失 > 30% |

**评审方法**: 调用 `getAssetCompletionStatus(projectId)` 获取完成率。

### 2. 一致性 (consistency)

同一角色在不同分镜中外貌是否一致，同一场景的色调是否统一。

| 分数 | 条件 |
|------|------|
| 1.0 | 无一致性问题 |
| 0.8 | 轻微风格波动，不影响观看 |
| 0.5 | 明显的角色外貌或场景风格不一致 |
| 0.0 | 严重的视觉跳变 |

**评审方法**: 调用 `getStoryboards(episodeId)` 查看分镜绑定的角色和场景，结合 `getCharacters(projectId)` 比对角色描述。

### 3. 技术质量 (technical_quality)

生成资产的技术状态是否正常。

| 分数 | 条件 |
|------|------|
| 1.0 | 所有资产状态正常 |
| 0.5 | 有少量资产状态异常但可用 |
| 0.0 | 大量资产 URL 为空或状态异常 |

**评审方法**: 调用 `getAssetCompletionStatus(projectId)` 检查缺失列表。

## 综合评分计算

```
综合分 = completeness × 0.5 + consistency × 0.3 + technical_quality × 0.2
```

## 关卡决策阈值

| 决策 | 条件 | 处理 |
|------|------|------|
| **PASS** | 综合分 ≥ 0.8 且无严重缺失 | 继续下一阶段 |
| **RETRY_SPECIFIC** | 0.6 ≤ 综合分 < 0.8 | targets 列出具体问题分镜 ID |
| **RETRY_ALL** | 综合分 < 0.6 | 整个阶段重新执行 |
| **ESCALATE** | 存在无法自动修复的问题 | 需要人工介入 |

## 按阶段的评审重点

### ASSET_IMAGE (图片质量)
- 重点: 完整性 + 角色一致性
- 检查: 是否所有分镜都有图片、同一角色在不同镜头中是否外貌一致

### ASSET_VIDEO (视频质量)
- 重点: 完整性 + 运动连贯性
- 检查: 是否所有分镜都有视频、视频时长是否符合分镜设定

### ASSET_AUDIO (音频质量)
- 重点: 完整性 + 音色匹配
- 检查: 是否所有有台词的分镜都有音频

### CONSISTENCY (跨镜头一致性)
- 重点: 风格一致性 + 角色一致性
- 检查: 整体视觉风格是否统一、角色形象是否一致
