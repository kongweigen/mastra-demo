# 角色去重策略

## 别名识别

| 模式 | 示例 |
|------|------|
| 姓氏+称谓 | "林小姐"→"林婉儿"、"张总"→"张明远" |
| 职务称呼 | "总裁"→根据上下文确定具体角色 |
| 昵称/简称 | "婉儿"→"林婉儿"、"老张"→"张明远" |
| 关系称呼 | "二婶"→"刘翠芬"（需结合上下文） |

**识别依据**：名称相似度 + relationships 关系网络 + 外貌描述相似度

## 合并规则

- **规范名选择**：完整姓名 > 出现频率高 > 更正式
- **字段合并**：逐字段比较，保留更详细的版本
- **缺失补充**：A 版本缺少 hair_style 但 B 版本有 → 用 B 的
- **冲突处理**：取描述更具体的（"白皙"优于"偏白"）
- **relationships 合并**：合并所有版本的关系信息，去重

## 合并示例

```
第1集: {"name": "二婶", "gender": "FEMALE", "clothing": ""}
第3集: {"name": "刘翠芬", "gender": "FEMALE", "clothing": "花棉袄", "face_features": "刻薄"}

合并后: {"name": "刘翠芬", "gender": "FEMALE", "clothing": "花棉袄", "face_features": "刻薄"}
nameMapping: {"二婶": "刘翠芬", "刘翠芬": "刘翠芬"}
```

## 输出格式

```json
{
  "characters": [
    {"name": "刘翠芬", "gender": "FEMALE", ...合并后完整字段...}
  ],
  "character_name_mapping": {
    "刘翠芬": "刘翠芬",
    "二婶": "刘翠芬",
    "翠芬婶子": "刘翠芬"
  }
}
```

- nameMapping 必须包含所有旧名→规范名的映射
- 规范名自身也映射到自身（确保下游查询一致性）
