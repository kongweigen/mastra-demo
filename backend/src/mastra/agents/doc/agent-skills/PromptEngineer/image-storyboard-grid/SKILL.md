# 宫格图 Prompt 工程规范

> **⚠ 强制规范 — 以下规则必须严格遵守，不可自行发挥：**
> 1. **必须使用中文**输出提示词（不可使用英文）
> 2. **NxN 宫格的每一格都必须有画面描述**（禁止留空、禁止"2 panels in 2x2"这种只填部分格子）
> 3. **必须使用规定格式**：`第N格（位置）：[镜头X+时间位置] 景别，画面描述`
> 4. **必须包含粗黑分隔线、一致性声明、风格后缀**
> 5. 违反以上任何一条都将导致后端裁剪失败

为多镜头组生成一张正方形网格图，所有格子填满画面内容（禁止留空）。镜头不足时按比例扩展，每个镜头占多个格子展现不同时刻。生成后由后端裁剪为每个镜头的首帧。

## 布局规则（必须正方形，禁止留空）

请求中会给出布局（如"布局: 3x3"），严格按此布局生成：

| 镜头数 | 布局 | 总格数 |
|--------|------|--------|
| 1-4 | 2x2 | 4 |
| 5-9 | 3x3 | 9 |
| 10-16 | 4x4 | 16 |
| 17-25 | 5x5 | 25 |

禁止非正方形布局（如 2x3、2x5、3x4 等），裁剪程序只支持正方形网格。

### 禁止空格——镜头扩展填充规则

**核心原则：每一格都必须有画面内容，绝不留空、绝不占位。**

当镜头数 < 总格数时，将总格数均匀分配给各镜头，每个镜头的剧情扩展为多个连续画面格：

**分配方法：** 总格数 ÷ 镜头数 = 每镜头基础格数（余数从第一个镜头开始依次 +1）。

| 镜头数 | 布局 | 分配方案 |
|--------|------|---------|
| 2 镜头 | 2x2(4格) | 镜头1→格1,2；镜头2→格3,4 |
| 3 镜头 | 2x2(4格) | 镜头1→格1,2；镜头2→格3；镜头3→格4 |
| 2 镜头 | 3x3(9格) | 镜头1→格1-5；镜头2→格6-9 |
| 3 镜头 | 3x3(9格) | 镜头1→格1-3；镜头2→格4-6；镜头3→格7-9 |
| 5 镜头 | 3x3(9格) | 镜头1→格1,2；镜头2→格3,4；镜头3→格5,6；镜头4→格7,8；镜头5→格9 |

**扩展画面的内容要求：**
- 同一镜头的多个格子必须展现该镜头剧情的不同时刻/角度/细节
- 例如：镜头1"二婶推门闯入"占2格 → 格1：二婶手推门板、门半开；格2：二婶已跨入门槛、手指前方
- 保持同一镜头各格之间的场景、光线、角色位置连贯
- 不同格子之间要有明显的动作进展或视角变化，不能简单复制同一画面

## Prompt 结构

```
[布局指令] + [各格内容] + [一致性] + [风格]
```

### 布局指令（固定前缀）

按请求中给出的布局写：
```
一张NxN宫格图，共N²个画面格，全部填满，用粗黑分隔线均匀分隔，白色背景。
```

示例（2镜头 → 2x2）：
```
一张2x2宫格图，共4个画面格，全部填满，用粗黑分隔线均匀分隔，白色背景。
```

### 各格内容

按阅读顺序（左到右，上到下）逐格描述，**每一格都必须有画面**：
```
（2镜头 → 2x2 示例：镜头1占格1,2；镜头2占格3,4）
第1格（左上）：[镜头1起始] 中景，二婶手推木门，门板半开，门外透光，昏暗卧室内。
第2格（右上）：[镜头1发展] 中景，二婶已跨入门槛，手指前方，表情凶狠，卧室全貌可见。
第3格（左下）：[镜头2起始] 中景，林晚双手张开挡在粮食袋前，眼神坚定。
第4格（右下）：[镜头2发展] 中景，二婶猛推林晚，林晚撞到墙上，粮食袋倾斜。
```

**格内容标注规则**：每格标注其所属镜头编号和时间位置（起始/发展/高潮/结束），方便裁剪后对应。

内容来源：从 getStoryboardDetails() 获取的 visual_prompt + 角色外貌 + 场景描述。

### 一致性
```
所有画面格中角色外貌保持一致，统一色调和光线。
```

### 风格
```
电影写实风格，整洁宫格布局，粗黑分隔线清晰，高细节，4K。
```

## promptParams（关键）

必须输出准确的 grid_rows 和 grid_cols，裁剪程序依赖这两个值：

```json
{
  "storyboardId": 请求中"分镜ID列表"的第一个纯数字ID（如 2035282211953319937）,
  "engineeredPrompt": "一张3x3宫格图...",
  "negativePrompt": "模糊, 低质量, 面部变形, 多余肢体, 水印, 格子合并, 分隔线不清晰, 空白格子, 留白",
  "promptParams": {"grid_rows": 3, "grid_cols": 3, "ratio": "1:1"}
}
```

grid_rows 和 grid_cols 必须相等（正方形），且与布局指令中的 NxN 一致。
ratio 固定为 "1:1"（正方形图片）。

## 首尾帧合并宫格

请求中含"首尾帧宫格"时，每个镜头生成首帧和尾帧两个画面，交替排列。

**同样禁止留空**——如果首尾帧总数 < 总格数，多余格子按扩展填充规则分配给各镜头（增加中间过渡帧）。

4 镜头首尾帧 → 需要 8 格 → 3x3（9格，多1格分配给镜头1）：
```
第1格：镜头1 开始画面（二婶站在门口，手推门板半开）
第2格：镜头1 过渡画面（二婶跨入门槛，快步向前）
第3格：镜头1 结束画面（二婶已冲到桌前，手按桌上）
第4格：镜头2 开始画面（林晚站在原地，表情惊恐）
第5格：镜头2 结束画面（林晚退到墙角，背靠墙壁）
第6格：镜头3 开始画面（林晚双手张开挡粮食袋）
第7格：镜头3 结束画面（林晚被推开，粮食散落）
第8格：镜头4 开始画面（二婶伸手准备推人）
第9格：镜头4 结束画面（林晚撞墙滑坐地上）
```

排列规则：基本为第(2i+1)格=镜头i首帧，第(2i+2)格=镜头i尾帧；多余格子插入过渡帧，从第一个镜头开始依次分配。
请求中会给出"镜头数: 4, 总格数: 8, 布局: 3x3"。

## 检查清单

- 布局必须正方形（NxN）
- grid_rows == grid_cols
- ratio 为 "1:1"
- **每一格都有画面内容，绝不留空**
- 镜头数 < 总格数时，已按扩展填充规则分配（每镜头占多格，展现不同时刻）
- 同一镜头的多格之间有动作进展或视角变化，不是简单复制
- 粗黑分隔线
- 角色外貌各格一致
- 中文提示词
