# Seedance 2.0 视频 Prompt 工程规范

## 模型能力

- 时长: 单镜头最短 1 秒，组总时长不超过 15 秒
- 分辨率: 720p / 1080p
- 画面比例: 16:9, 9:16, 1:1
- 原生音频: generate_audio=true（环境音+配音+配乐）
- 多模态输入: 参考图片、参考视频、参考音频
- 多语言: 中文、英文等

## 多模态参考素材

后端自动将参考素材放入 content 数组，请求中的"参考素材信息"已给出编号映射。

content 数组顺序:
1. 角色参考图 (有设计图的角色)
2. 场景参考图 (有设计图的场景)
3. 宫格图 (如有)
4. 角色音色文件 (只包含本组有台词的角色，按首次出场顺序)
5. 旁白音色文件 (如有旁白台词)

prompt 中用 @ 引用:
- "@图片1中的XX" "@图片2中的XX"（角色/场景参考图）
- "使用@音频N的声线"（音色参考文件）

编号严格对应请求中"=== content 数组编号 ==="部分给出的映射。

## 台词信息

请求中"=== 各镜头台词 ==="已列出每个镜头的说话人和台词内容。
不需要再调用 getStoryboardDetails。

重要规则:
- 角色对话: 将对话自然融入动作描述，指定"使用音频N的声线"
- 旁白台词: 不写进 prompt（由独立 TTS 生成），但旁白镜头的画面和动作仍需描述
- 无台词角色: 请求中会标注"角色XX在本组镜头中无台词"，这些角色不需要指定音色

## Prompt 写法

### 多镜头（一次调用生成多个镜头）

通过时间段描述实现多镜头切换:

```
0-3秒: 全景，昏暗的林家卧室，@图片1中的年轻女性蜷缩在床角，@图片2中的中年女性站在门口。
3-5秒: 中景，中年女性一脚踹开门冲进来，手指着年轻女性的鼻子。中年女性使用@音频1的声线。
5-7秒: 特写，年轻女性瞪大双眼，不自觉地后退半步。
7-10秒: 中景，年轻女性双手张开挡在粮食袋前，嘴唇发抖但眼神坚定。年轻女性使用@音频2的声线。
10-13秒: 中景，中年女性猛推年轻女性，粮食袋散落一地。画面风格与@图片3的场景一致。
13-15秒: 特写，油灯火苗在两人之间摇曳，旁白低沉地叙述。旁白使用@音频3的声线。
```

一组可以包含 5-7 个镜头，单个镜头可短至 1 秒，总时长不超过 15 秒。简单动作用短镜头（1-3 秒），有台词的用稍长镜头。

### 单镜头（组内只有1个成员）

不用时间线分段，直接描述整个画面:

```
在昏暗的林家卧室中，@图片1中的年轻女性猛地从床上坐起，额头满是冷汗。她双手颤抖地抓住被单，环顾四周。窗外传来凛冽的寒风声，油灯微光在墙壁上摇曳。镜头从中景缓缓推到特写，捕捉她眼中的恐惧与决心。整体场景氛围与@图片2一致。角色使用@音频1的声线。
```

### 旁白镜头（旁白台词写进 prompt）

Seedance 2.0 支持 generate_audio，旁白台词写进 prompt，使用旁白音色生成语音，音画更同步:

```
零下八十度的暴风雪中，几个模糊人影在黑暗中疯狂抢夺物资，煤块散落积雪。镜头缓慢平移记录末日般的混乱。旁白低沉地叙述。旁白使用@音频3的声线。整体风格真实冷冽，电影感光影。
```

注意: 旁白台词不要逐字写进 prompt（否则太长），而是用"旁白低沉地叙述""旁白缓缓道来"等概括性描述融入画面。旁白音色通过"使用@音频N的声线"指定。

### 角色对话 + 旁白（融入动作描述）

角色说话和旁白都自然融入动作描述中:

```
@图片1中的中年女性冲进来，声嘶力竭地质问，手指着对面。@图片2中的年轻女性双手护住粮食，坚定地回应。中年女性使用@音频1的声线，年轻女性使用@音频2的声线。旁白在画外低声叙述，使用@音频3的声线。
```

## 音频控制

- 环境音自然融入描述（"狂风呼啸""门被踹开的碰撞声"）
- 角色声线通过"使用@音频N的声线"指定（N 对应编号映射）
- 旁白声线也通过"使用@音频N的声线"指定（N 对应旁白的音色编号）
- 背景音乐通过描述控制（"背景音乐节奏紧张""轻柔钢琴旋律"）
- 只有在编号映射中出现的角色才指定音色，无台词角色不指定

## 输出格式

prompts 数组只有一个元素，storyboardId 填第一个分镜 ID:

```json
{
  "prompts": [
    {
      "storyboardId": 100,
      "engineeredPrompt": "0-4秒: ...\n4-8秒: ...",
      "negativePrompt": "",
      "promptParams": {"duration": 12, "ratio": "9:16"}
    }
  ],
  "summary": "生成了1条组视频 prompt，12s"
}
```

## 关键规则

- prompt 中用"@图片N""@音频N"引用参考素材，编号严格对应请求中的映射
- 自然语言叙事风格，不用特殊标记语法
- 多镜头用"N-M秒:"分段，单镜头不分段
- 旁白台词概括性融入 prompt（"旁白低沉地叙述"），不逐字写入，指定旁白音色
- 角色对话融入动作描述，通过"使用@音频N的声线"指定音色
- 首次提到角色时写"@图片N中的XX"，后续直接用角色名
- 每条 prompt 控制在 1500 字符内
- 时长限制: 组总时长不超过 15 秒，台词总量也需控制在 15 秒内能说完（台词过多时精简或概括）
- 无台词角色不指定音色

## 实战技巧

- 每组第一个镜头用全景描述人物站位（谁在画面左侧/右侧），避免轴线错乱
- 人物面朝方向要明确: "人物面朝画面左侧"
- 简单对话不要过度描述景别，让模型自由运镜会更自然
- 写"镜头转换为..."比"切镜头"效果更好，避免生硬切换
- 复杂连续动作（打斗、舞蹈）可以简洁描述让模型自由发挥
- 想让画面多停留: 在末尾加动作描述（"陷入沉思""目光凝视远方"）
- 3秒能完成的动作不要给5秒，否则会出现慢动作
- 多人场景（3人以上）需要详细描述站位，否则容易站位错乱
- 单人或双人镜头可以由模型自由发挥
