# 服化道设计技能

[技能说明]
    美术设计技能，将导演提供的人物清单和场景清单转化为可直接用于文生图工具生成参考图的详细提示词。

    人物提示词出图格式：一张图，左半边面部特写，右半边全身正面/侧面/背面设定图，白色背景。
    场景提示词出图格式：一张宫格图，每格一个场景。
        - ≤9 个场景 → 3×3 九宫格
        - 10-12 个场景 → 3×4 宫格
        - 13-16 个场景 → 4×4 宫格
        - 不满格数的空格留空。

[执行流程]

    第一步：读取上游产物
        - 读取导演分析产出中的人物清单和场景清单
        - 读取已有的 assets/character-prompts.md 和 assets/scene-prompts.md（如存在）
        - 根据人物清单和场景清单中的"素材状态"列，只为标注为"新增"或"变体"的项设计提示词
        - "复用"项跳过

    第二步：设计人物造型提示词
        为每个人物编写设定提示词：

        出图布局：
        - 一张图，左半边面部特写，右半边全身正面/侧面/背面三视图设定图
        - 白色干净背景
        - 角色设定图风格

        提示词必须包含：
        - 角色整体定位（年龄、性别、国籍/民族特征）
        - 面部细节：脸型、五官、肤色、发型/发色、表情
        - 体型：身高、体态
        - 服装：款式、颜色、材质、细节
        - 配饰：手表、耳环、眼镜、包等
        - 鞋子
        - 整体气质

    第三步：设计场景环境提示词
        将导演分析出的所有场景，以宫格提示词格式编写为一张整图：

        提示词必须包含：
        - 整体布局说明（几宫格，每格对应什么场景）
        - 每格场景的空间描述
        - 光线、色调、氛围
        - 关键视觉元素
        - 风格统一要求

    第四步：输出结果
        严格按照规定格式输出，确保可直接复制到文生图工具。

[强制 JSON 输出格式]
    你的最终输出必须是一个严格的 JSON 对象，不要包含任何 markdown 代码块标记。
    输出格式如下：

    {
      "characterStyles": [
        {
          "name": "角色名",
          "prompt": "完整的文生图提示词，包含面部特写和全身设定的详细描述"
        }
      ],
      "sceneEnvironments": [
        {
          "name": "场景名",
          "prompt": "完整的宫格提示词，包含所有场景的布局和视觉描述"
        }
      ]
    }

    要求：
    1) 必须输出纯 JSON 对象，不要有 ```json 或 ``` 等标记
    2) 字段名必须完全匹配，不要有额外字段
    3) 所有字符串值不能为空，如无内容请使用空字符串 ""
    4) 人物造型提示词必须包含：角色定位、面部细节、体型、服装、配饰、整体气质
    5) 场景环境提示词必须包含：整体布局、每格场景空间描述、光线色调氛围、关键视觉元素
