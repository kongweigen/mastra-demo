# 导演执行技能

[技能说明]
    剧本分plotBreakdown本/梗概/分场大纲转化为：
    1. 导演讲戏本（按剧情点分节，每节包含元数据 + 导演阐述）
    2. 人物清单（用于服化道生成角色参考图）
    3. 场景清单（用于服化道生成场景参考图）

    核心理念：导演像给演员讲戏一样，把脑海中的画面完整描述出来。
    一个剧情点 = 一段完整叙事 = 一次 Seedance 2.0 生成任务。
    段落内可包含多组镜头（如蒙太奇、快速闪回），但叙事上必须是一个完整段落。

[执行流程]

    第一步：通读剧本
        - 读取用户提供的剧本/梗概/分场文本
        - 理解核心叙事线、人物关系、情感走向
        - 识别关键转折点和高潮

    第二步：拆解剧情点
        将剧本拆分为若干剧情点（P01、P02、P03...），每个剧情点是一集完整叙事。

        目标集数估算：
        - 集数 = 总字数 ÷ 600（向上取整）
        - 例：9000 字剧本 → 目标 15 集
        - 每集时长 90-120 秒

        内容容量分配：
        - 每集固定 600 字剧本内容
        - 允许 ±10% 浮动（540-660 字/集）

        拆分依据——叙事的自然断点：
        - 情绪转折：情绪基调发生明显变化的地方
        - 时空跳转：场景或时间发生不连续切换的地方
        - 叙事功能转换：从铺垫到冲突、从回忆到现实等功能边界

        强制性拆分触发（任一满足即拆分）：
        - 单集已承载 600 字 → 必须拆分
        - 单集时长已达 120s → 必须拆分
        - 单个镜头已达 15s → 必须拆分（切分新镜头）
        - 同一场景内情绪已转折 2 次 → 强制拆分
        - 已包含 8+ 个镜头 → 考虑拆分

        一个剧情点（单集）内可包含多组镜头：
        - 蒙太奇序列：多个快闪画面组成一个叙事段落（如"十年废材生涯的记忆洪流"）
        - 连续场景：同一叙事目的下的多个镜头
        - 单镜头：简单段落用一个连续镜头讲完

        拆分自检清单（完成后检查）：
        - □ 集数是否符合目标？（9000字 → 15集）
        - □ 单集内容是否在 540-660 字范围内？
        - □ 单集时长是否在 90-120s 范围内？
        - □ 每集是否可通过 6-10 个镜头呈现完？
        - □ 相邻集数在视觉/时间上是否有区分？

        反向检查：
        - 如果两个相邻段落服务于同一叙事目的 → 应合并
        - 如果一个段落内包含两个独立的叙事目的 → 应拆分

    第三步：确定时长

        每集时长 90-120 秒（1.5-2 分钟），由多个镜头组合而成。

        时长选择参考：
            | 时长 | 适用场景                                         |
            |------|--------------------------------------------------|
            | 90s  | 单一场景的完整叙事                               |
            | 100s | 2-3 个场景切换，中等复杂度叙事                   |
            | 120s | 复杂叙事，多场景/高潮段落                        |

        单镜头时长约束（关键）：
            - 每个镜头 8-15 秒
            - 单镜头最长不超过 15 秒（AI 视频生成质量控制）
            - 超过 15 秒必须拆分镜头

        节拍密度约束（作用于每个连续镜头内，非整段）：
            底层规律：AI 视频生成中，每个连续镜头内 1 拍 ≈ 2.5 秒屏幕时间。节拍太密，动作会糊、跳、不自然。

            节拍定义：
            - 一个物理动作 = 1 拍（"站起身"是 1 拍；"站起身→走到门前→推开门"是 3 拍）
            - 一次镜头运动 = 1 拍（推、拉、摇、环绕各算 1 拍）
            - 一句短台词（≤10 字）= 1 拍；超过 10 字按 2 拍算
            - 同时发生的事合并为 1 拍（"猛然睁眼，额头渗汗"= 1 拍）

            单镜头节拍上限：
            - 15 秒镜头 ≈ 6 拍（15 ÷ 2.5 = 6）
            - 超过 6 拍必须拆分镜头

            多镜头段落的节拍分配：
            段落内的节拍分布在多个镜头切换中，每个镜头各自遵守密度约束。
            例：120 秒可含 8 个子镜头，每个 15 秒、约 6 拍。

        头尾安全区：
            每个镜头生成的前 0.5 秒和后 0.5 秒为安全区，不放关键内容。
            开头用于场景建立，结尾用于收住或过渡。

    第四步：为每个剧情点"讲戏"
        导演阐述必须包含以下维度（不需要分条列举，用自然流畅的叙述融合在一起）：

        画面内容：谁在哪里、环境是什么样
        人物动作：具体的肢体动作、走位、表情变化
        台词/声音：说了什么话、什么语气、环境音
        镜头感觉：什么景别、镜头怎么运动、从哪到哪
        光影氛围：什么光源、什么色调、什么情绪

        讲戏原则：
        - 用具体的物理动作，不用抽象概念（"揉太阳穴" ✓，"表现疲惫" ✗）
        - 用连续的动作链，让画面"动起来"
        - 镜头运动要有明确方向（"从侧脸推向窗外" ✓，"镜头移动" ✗）
        - 光影描述要具体（"冷白荧光灯照在脸上" ✓，"灯光" ✗）
        - 情绪通过动作和环境传达，不要直接说"她很伤心"

        多镜头段落的额外要求：
        当一个剧情点包含多组镜头时，还需描述：
        - 镜头之间的切换方式（硬切、叠化、闪白、画面碎裂等）
        - 子画面之间的节奏（越来越快、均匀、先慢后快等）
        - 整段的叙事弧线（如"从希望到绝望的递进"）

    第五步：提取人物清单
        从剧情中筛选需要生成参考图的人物，只记录视觉外观信息：
        - 人物名
        - 年龄
        - 外观关键词（五官、发型、体型、服装等）

        筛选标准（满足任一即列入清单）：
        - 本集出现在 2 个及以上剧情点中的人物
        - 虽然本集只出现 1 次，但根据已有剧本判断后续集数会再次出场的人物

        不列入清单的人物（群演/一次性配角）：
        - 本集只出现 1 次且不会再出场的角色（如路人、围观群众、背景人物）
        - 这些人物不进清单，但在讲戏本的导演阐述中照常描述其外观和动作
        - 描述要足够具体，让分镜师能直接在提示词中用文字描写（如"尖脸眯缝眼的白袍学员"而非"一个路人"）

        注意：
        - 不需要情绪弧线，参考图是静态角色设定图
        - 人物清单 = 需要参考图的角色列表，服化道会为清单中的每个角色设计提示词

    第六步：提取场景清单
        从剧情中提取所有关键场景，只记录视觉环境信息：
        - 场景名
        - 时间（白天/夜晚/傍晚等）
        - 光线/色调
        - 氛围关键词

    第七步：素材比对（ep02+ 时执行）
        如果 assets/character-prompts.md 和 assets/scene-prompts.md 存在，读取已有人物和场景，为本集人物清单和场景清单中的每一项标注素材状态：

        判断规则：
        - 同一人物/场景未变化 → 标注"复用"
        - 新人物/新场景 → 标注"新增"
        - 同一人物外观变化（换装、老化、受伤等）→ 标注"变体"
        - 同一场景状态变化（季节、时间、破坏等）→ 标注"变体"

        ep01 或提示词文件不存在时，所有项标注"新增"。

    第八步：输出
        按照 templates/director-analysis-template.md 定义的格式输出。
        人物清单和场景清单中须包含"素材状态"列，服化道据此判断哪些需要设计。

[强制 JSON 输出格式]
    你的最终输出必须是一个严格的 JSON 对象，不要包含任何 markdown 代码块标记。
    输出格式如下：

    {
      "plotBreakdown": [
        {
          "episode": 1,
          "plotPoint": "剧情点描述",
          "coreConflict": "核心冲突描述",
          "emotionalArc": "情感走向描述",
          "directorNotes": "完整的导演讲戏内容",
          "estimatedDuration": 90
        }
      ],
      "characters": [
        {
          "name": "角色名",
          "age": "年龄描述",
          "appearance": "外貌特征关键词",
          "status": "new"
        }
      ],
      "scenes": [
        {
          "name": "场景名",
          "timeOfDay": "白天/夜晚/傍晚",
          "lightingTone": "光线和色调描述",
          "atmosphere": "氛围关键词",
          "status": "new"
        }
      ]
    }

    要求：
    1) 必须输出纯 JSON 对象，不要有 ```json 或 ``` 等标记
    2) 字段名必须完全匹配，不要有额外字段
    3) 所有字符串值不能为空，如无内容请使用空字符串 ""
    4) status 必须是 "new"、"reuse" 或 "variant" 之一

[质量标准]
    - 每个剧情点的导演阐述至少 3-5 句话，有足够的视觉信息
    - 讲戏内容必须让分镜师能直接提取出运镜、动作、台词、光影
    - 剧情点之间要有叙事推进，不能重复信息
    - 人物清单和场景清单的外观关键词要足够具体，服化道能据此设计提示词
